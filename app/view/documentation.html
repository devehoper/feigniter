Feigniter Docs!

Your guide to building awesome websites, made super simple!

Imagine you want to build a super cool treehouse. Feigniter is like a giant box of special LEGOs just for building websites! Instead of starting from scratch with wood and nails, you get lots of ready-made pieces that fit together perfectly. This makes building your website faster and way more fun.

Routing is like a map for your website. When you type a URL with a # symbol, Feigniter looks at the map to see which page to show you. 
#HomeController?about
It's like telling your GPS, "Go to the **HomeController** house and show me the **about** room!"

You can even give it special instructions: 
#UserController?showProfile=123
This tells the **UserController** to run the **showProfile** method and give it the number **123**.

A Controller is the "brain" for each page. If your webpage were a robot, the Controller would be the computer inside that tells its arms and legs what to do. It handles all the logic, like what happens when you click a button. 
// app/controller/MyPageController.js
export default class MyPageController extends Controller {
    index() {
        // This is the brain of your page!
        // It tells the app to load the 'face' (the View).
        this.loadView('app/view/mypage.html');
    }
}

A View is the "face" of the page. It's the HTML code that decides what the page looks like‚Äîwhere the pictures go, what the text says, and what color the buttons are. The Controller (the brain) loads the View (the face) to show it to the user. 
<!-- app/view/mypage.html -->
<h1>Welcome to My Page!</h1>
<p>This is the face of the page.</p>
<button>Click me!</button>

The config.js file has all the default settings, like a new video game right out of the box. You should **never change this file!**

Instead, you make your changes in userConfig.js. Think of userConfig.js as your own special settings file that tells the game, "Hey, I want to play on hard mode!" or "I want my character to wear a blue hat!". Your settings in userConfig.js will always win. 
// In userConfig.js
const userConfig = {
    homeController: "MyCoolHomeController",
    debugMode: true // Turn on "easy mode" to see helpful messages!
};

A Model helps you manage your data. One of its most important jobs is to be a **Rule Checker**. Before you save any information (like a username or email), you give it to the Model with a list of rules. The Model checks if the data follows all the rules. If it doesn't, it tells you exactly what's wrong! 
// In your controller...
const userData = { email: "not-an-email", age: 15 };

const rules = {
    email: { required: true, email: true },
    age: { required: true, min: 18 }
};

const errors = Model.validateData(userData, rules);

if (Object.keys(errors).length > 0) {
    // Oops, there are errors!
    // errors will be: { email: "Invalid email format", age: "age must be at least 18" }
    Model.displayValidationErrors(errors);
}

Feigniter has a magic translator! You can create language files (like en.json for English and pt.json for Portuguese). When a user picks a language, Feigniter automatically looks in the right file and shows the correct words on the page.

Caching is like having a super-fast memory. The first time you visit a page, Feigniter saves a copy of it. The next time you come back, it just shows you the saved copy instantly instead of building it all over again. It's why pages load so quickly!

Before you show your website to the whole world (this is called "going to production"), you want to make it as fast and safe as possible. It's like cleaning up your treehouse before your friends come over!

In your userConfig.js file, you should:
  • Turn off "easy mode" (debugMode: false) so no secret messages are shown.
  • Turn on the "super-fast memory" (useCache: true) to make your site load instantly for visitors.
// In userConfig.js for production
const userConfig = {
    debugMode: false,
    useCache: true
};

Before you show your treehouse to everyone, you want to make sure it's safe. You have a special robot helper (named Jest) that can test your code for you!

When you run the test command, the robot will automatically check all your special LEGO pieces to make sure they work correctly. If it finds a problem, it tells you exactly where it is so you can fix it. 
npm test
To use this robot, you need Node.js installed on your computer (see the next topic!).

Think of Node.js as your personal workshop. Your website (the treehouse) doesn't need the workshop to stand and be used by your friends. But **you** need the workshop to use your cool power tools!

In this project, Node.js is used for two main power tools:
  • Testing: It runs your test robot (Jest) to make sure your code is strong and bug-free.
  • Building for Production: It runs a "shrink-ray" that makes your code smaller and harder to copy before you put it on the internet.
So, you only need Node.js on your own computer to build and test the website. Your website's visitors don't need it at all!
